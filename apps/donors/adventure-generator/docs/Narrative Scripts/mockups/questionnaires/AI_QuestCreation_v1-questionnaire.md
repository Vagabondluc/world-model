# UI Questionnaire: AI Storyteller Dashboard (Quest Creation)

**Version:** 0.1.0
**Status:** Draft
**Owner:** TBD
**Last Updated:** 2026-02-04


## 1. Primary Purpose
A narrative-orchestration interface designed for AI-driven storytelling where the AI acts as the "Storyteller" (GM) and the user as the "Protagonist," facilitating interactive quest development and session management.

## 2. Core Inputs
- **Narrative Premise/Starting Point:** (e.g., "A deserted freighter in deep space").
- **Protagonist Profile:** (Character traits, intentions, and key abilities).
- **Core Setting Parameters:** (Rules for consistency, established context).
- **Conflict Initializer:** (The primary threat or obstacle).

## 3. UI Requirements
- **Interactive Story Feed:** A dual-stream chat/log where Storyteller descriptions and Protagonist actions are clearly differentiated.
- **Narrative Logic Toggle:** A tool for the Storyteller (AI) to evaluate tasks as "Effortless" vs "Complex," with visible DC/Difficulty indicators for the user.
- **Environment Context Monitor:** A persistent panel showing the "Established Context" (e.g., "Deserted freighter, no police, rogue androids present") to ensure consistency.
- **Conflict Resolution Console:** A specialized UI for managing turn-based sequences, tactical positioning, and outcome descriptions.
- **Sensory Prompt Generator:** A button to expand a "Subjective Experience" description (Sights, Sounds, Internal feelings).

## 4. Derived & Automated Fields
- **Consequence Engine:** Automatically suggests outcome options based on the protagonist's described intentions and character abilities.
- **Consistency Auditor:** Flags actions or descriptions that contradict the "Environment Context" or established lore.
- **Subjective Layering:** Automatically injects "Subjective experience" and "background knowledge" into standard descriptions.

## 5. Exports & Integration
- Session Transcript (Markdown).
- Summarized Quest Synopsis.
- Character Experience & Growth Log.
