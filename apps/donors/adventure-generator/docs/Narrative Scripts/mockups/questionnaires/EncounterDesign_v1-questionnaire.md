# UI Questionnaire: EncounterDesign_v1

**Version:** 0.1.0
**Status:** Draft
**Owner:** TBD
**Last Updated:** 2026-02-04


## 1. Primary Purpose
A streamlined, high-efficiency encounter builder optimized for mid-session prep and easy tabletop reference.

## 2. Core Inputs
- **Scene Snapshot:** Brief, high-impact description.
- **Core Goal:** One-sentence objective.
- **Environment Hack:** Single rule/modifier for the terrain.
- **Challenge Steps:** Bulleted instructions for skill checks/puzzles.
- **Fast Consequences:** Short "If Win/If Lose" bullets.
- **Statblock Reference:** Succinct opponent stats and 1-2 key tactics.
- **XP Total:** Clear sum for all components.

## 3. UI Requirements
- **Fast-Build Form:** Minimized fields with placeholder examples to speed up entry.
- **Scannable Layout:** Information organized into "Cheat Sheet" blocks.
- **Tactics Bullet Points:** A dedicated area for "When [X] happens, NPC does [Y]".
- **Auto-Cumulative XP:** A running total that stays visible.

## 4. Derived & Automated Fields
- **Hook Suggestor:** Direct, one-line transitions to the next scene.
- **CR-to-XP Mapper:** Automates XP based on monster selection.

## 5. Exports & Integration
- Export to "Compact Encounter Card" (Index card size).
- Markdown Snippet.
