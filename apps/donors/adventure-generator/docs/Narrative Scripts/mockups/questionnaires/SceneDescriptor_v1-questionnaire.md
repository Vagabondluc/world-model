# UI Questionnaire: SceneDescriptor_v1

**Version:** 0.1.0
**Status:** Draft
**Owner:** TBD
**Last Updated:** 2026-02-04


## 1. Primary Purpose
A narrative tool to generate immersive, vivid scene descriptions for roleplaying games, providing both player-facing atmosphere and GM-facing mechanics.

## 2. Core Inputs
- **Genre/Setting:** (e.g., Cyberpunk, Gothic Horror, High Fantasy).
- **Location/Situation:** The physical space or immediate event.
- **Characters/Creatures/Objects:** Specific entities present in the scene.
- **Goals/Challenges/Events:** The primary motivation or hurdle in the scene.
- **Atmospheric Details:** Sensory cues (smells, sounds, lighting) to set the mood.

## 3. UI Requirements
- **Scene Context Panel:** Fields for Genre, Location, and specific Entities.
- **Narrative Focus Editor:** Inputs for the primary Goals/Challenges.
- **Atmosphere Lab:** A dedicated area for inputting sensory details (Sight, Sound, Smell, etc.).
- **Live Preview Window:** A split-view showing the "Player-Facing Description" (Vivid) and the "GM Notebook" (Running the scene).
- **Tone Slider:** A control to adjust the "Temperature" or intensity of the prose.

## 4. Derived & Automated Fields
- **Vivid Prose Generator:** Expands bullet points into immersive paragraphs for the players.
- **GM Strategy Suggester:** Automatically derives "Key Points for Running" based on the Goals and Challenges provided.
- **Entity Interaction Highlights:** Flags characters or objects for specific player interaction prompts.

## 5. Exports & Integration
- Scene Description Page (Markdown).
- GM Reference Card.
- VTT Note / Handout.
