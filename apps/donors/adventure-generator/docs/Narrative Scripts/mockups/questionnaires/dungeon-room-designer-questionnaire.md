# UI Questionnaire: dungeon-room-designer

**Version:** 0.1.0
**Status:** Draft
**Owner:** TBD
**Last Updated:** 2026-02-04


## 1. Primary Purpose
A detailed "Room Forge" that ensures every room has sensory depth, interactive elements, and logical player interactions.

## 2. Core Inputs
- **Room Type:** (Scenic, Challenge, Combat, Twist, Reward).
- **Physical Traits:** Shape, Size, Architecture.
- **Interactive Elements:** (Min 3 required).
- **Sensory Palette:** (Sight, Sound, Smell, Touch, Taste).
- **"Flavor" Details:** 2 irrelevant-but-cool world-building elements.
- **Mechanics:** Expected skill checks and enemy tactics.

## 3. UI Requirements
- **Progressive Disclosure Form:** Elements appear as you select the Room Type.
- **Sensory Meter:** A visual indicator that fills up as you address different senses.
- **Interaction Planner:** A table to map "Player Action" -> "Room Reaction".
- **AI Narrative Draft:** Instant transition of fields into an evocative description.

## 4. Derived & Automated Fields
- **Irrelevant Detail Suggestor:** Randomly pulls from a "Curiosity Table" to fill the 2 extra details.

## 5. Exports & Integration
- Export to Dungeon Key Writer.
- Standalone Room Card (Markdown).
