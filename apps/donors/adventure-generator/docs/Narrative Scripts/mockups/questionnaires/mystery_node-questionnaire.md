# UI Questionnaire: mystery_node

**Version:** 0.1.0
**Status:** Draft
**Owner:** TBD
**Last Updated:** 2026-02-04


## 1. Primary Purpose
The atomic building block of a mystery scenario: a detailed, interactive scene designed to provide clues, challenges, and narrative momentum through "Bangs" and layered discovery.

## 2. Core Inputs
- **Mystery Name & Node Code:** (e.g., "The Stolen Heart", "Node B").
- **Node Purpose/Stakes:** What is the "Agenda" of this scene?
- **Setting Details:** Vivid physical description (utilizing the "Three of Five" senses technique).
- **Characters:** Categorization of Leads, Features, and Extras.
- **Clues:** At least three distinct pieces of evidence (Physical, Testimony, Environmental).
- **Challenges:** Obstacles requiring skill checks or tactical choices.
- **"The Bang":** An explosive event to kick off the scene.

## 3. UI Requirements
- **Scene Framing Console:** Fields for Setting, Agenda, and the "Bang" event.
- **NPC Dossier Cards:** Profiles with motivations, dialogue prompts, and reaction logic.
- **Clue Repository:** A structured list for tracking clues and their outbound leads to other nodes.
- **Atmosphere Visualizer:** Tools to define mood, lighting, and soundscape (Three-by-Three method).
- **Transition Planner:** Options for "Sharp Cuts" or "Abstract Time" to the next scene.

## 4. Derived & Automated Fields
- **Sense Prompt Generator:** Automatically suggests "Three of Five" sensory details based on the setting.
- **Difficulty Calculator:** Suggests skill check DCs based on the node's position in the mystery hierarchy.
- **Clue Distribution Auditor:** Ensures a mix of obvious and subtle evidence.

## 5. Exports & Integration
- 1x Individual Mystery Node File (.txt).
- Printable Scene Reference Card.
- VTT Active Scene Overlay.
