# UI Questionnaire: node_cloud_mystery

**Version:** 0.1.0
**Status:** Draft
**Owner:** TBD
**Last Updated:** 2026-02-04


## 1. Primary Purpose
A heavy-duty non-linear investigation engine designed for open-ended "Node Cloud" scenarios (10-20 nodes), supporting player-driven exploration and robust clue distribution.

## 2. Core Inputs
- **Mystery Scope:** City-wide, planetary, multi-faction, etc.
- **Node Count:** Total number of nodes (10-20 recommended).
- **Core Mystery Elements:** The crime, perpetrator, and motive.
- **Node Registry:** List of diverse locations, people, and organizations.
- **Reactive Nodes:** 2-3 nodes that change based on player interaction.
- **NPC Network:** A list of NPCs with unique goals and faction ties.

## 3. UI Requirements
- **Interactive Cloud Map:** A zoomable "constellation" view of 10-20 nodes and their many inter-connections.
- **Redundancy Tracker:** A tool to verify "The Three Clue Rule" across the entire cloud (ensuring critical intel has 3+ sources).
- **Temporal Timeline Manager:** A dynamic timeline showing events that unfold if the players don't intervene.
- **Faction Influence Layer:** A toggle to see which factions control or influence specific nodes.
- **Reactive State Editor:** Controls for defining how nodes evolve (e.g., "If the warehouse is burned, the harbor node becomes alert").

## 4. Derived & Automated Fields
- **Entry Point Suggester:** Recommends the 3-5 best starting nodes based on clue density.
- **Clue Flow Simulator:** Visualizes how players might navigate the cloud to solve the core mystery.
- **Orphan Detector:** Flags any node that doesn't have at least three connections.

## 5. Exports & Integration
- 10-20 Individual Cloud Node Files (.txt).
- NPC Network Matrix.
- Faction Interest Report.
- Reactive Node Logic Doc.
- GM Operation Hub (Markdown).
