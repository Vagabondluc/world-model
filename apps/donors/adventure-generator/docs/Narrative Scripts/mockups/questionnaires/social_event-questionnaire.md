# UI Questionnaire: social_event

**Version:** 0.1.0
**Status:** Draft
**Owner:** TBD
**Last Updated:** 2026-02-04


## 1. Primary Purpose
A dynamic social event engine designed to manage complex guest lists, shifting conversation topics, and unfolding event sequences in a holistic urban environment.

## 2. Core Inputs
- **Holistic Location:** Zones of activity (e.g., The Balcony, The Ballroom, The Library).
- **Guest List (10-20 NPCs):** Names + Universal Roleplaying Template details.
- **Event Sequence:** Linear or "Grab Bag" major events (e.g., "The Duke arrives", "A glass shatters").
- **Conversation Topics:** Political gossip, recent news, trivial items.
- **Opening Hook:** The immediate scene-setter.

## 3. UI Requirements
- **Zone-Based Guest Tracker:** A visual map of "Who is in which room?" with drag-and-drop movement.
- **Topic Heatmap:** Shows which NPCs are discussing which topics in real-time.
- **Event "Grab Bag" Buttons:** A panel of available events that can be triggered with one click.
- **Handout Generator:** Automatically creates "Face Cards" or "Calling Cards" for the top 5 relevant NPCs.

## 4. Derived & Automated Fields
- **Relationship Map:** A spider-graph showing ties between the 20 NPCs on the guest list.
- **Pacing Monitor:** Flags if a social group hasn't changed location/topic in over 15 minutes.

## 5. Exports & Integration
- Social Event Master Sheet (PDF).
- NPC "Flash Cards" for players.
