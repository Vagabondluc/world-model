# UI Questionnaire: vivid_description

**Version:** 0.1.0
**Status:** Draft
**Owner:** TBD
**Last Updated:** 2026-02-04


## 1. Primary Purpose
A prose-enhancement tool that transforms bland, functional scene descriptions into vibrant, immersive, and sensory-rich narratives for RPG play.

## 2. Core Inputs
- **Bland Description:** The raw, functional text to be upgraded (e.g., "The players enter a cave").
- **Sensory Focus:** (Sight, Sound, Smell, Taste, Touch - which to emphasize).
- **Desired Tone/Emotion:** (e.g., Dread, Awe, Relief, Grief).
- **Urgency Level:** (Low to High - affects sentence length and verb choice).
- **Key Entities/Objects:** Specific items that need "intricate detail."

## 3. UI Requirements
- **Input/Output Transformer:** A side-by-side view showing the original "Bland" text and the generated "Vivid" text.
- **Sensory Layer Toggles:** Checkboxes to ensure all five senses are represented in the final prose.
- **Tone & Emotion Matrix:** A selection grid to inject emotional depth (Memories, Solace, Fear).
- **Action Verb Injector:** A tool to automatically swap passive verbs for "Strong Action Words" (e.g., "The wind whips" instead of "It is windy").
- **Figurative Language Suite:** Toggles for Similes, Metaphors, and Personification.

## 4. Derived & Automated Fields
- **Believability Auditor:** Ensures sensory details remain consistent (e.g., if it's sunset, the lighting matches).
- **Sentence Pacing Manager:** Automatically shortens sentences when the "Urgency" slider is high to build tension.
- **Interactive Prompt Generator:** Suggests ways for the DM to "Encourage Sensory Exploration" (e.g., "Invite the player to touch the bark").

## 5. Exports & Integration
- Immersive Scene Text (Markdown).
- Narrator's Reading Script.
- VTT Ambient Description Note.
