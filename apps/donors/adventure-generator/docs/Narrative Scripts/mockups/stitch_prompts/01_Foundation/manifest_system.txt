## 2. TECHNICAL SPECIFICATIONS (LOGIC & STYLE)
# UI Explanation: Manifest Explorer (index/manifest system)

## Component Architecture
- **Hierarchical Repository Tree:** A folder-based navigation system derived from `index.json`. It maps the physical file structure of `Narrative Scripts` to a user-friendly UI.
- **Unified Catalog Reader:** An interface that resolves scripts across `manifest.json` and `manifest.yaml` simultaneously, abstracting the format differences for the user.
- **Dependency Map Panel:** A dedicated section showing the inputs (requirements) and outputs (produced artifacts) of a selected script.
- **Artifact Registry:** A list of all files generated by the selected tool in the current active workspace.

## Interaction Logic
- **Global Path Validation:** A maintenance tool that checks if all file paths in the manifests still point to valid files on disk.
- **Auto-Discovery Toggle:** When "Scan Root" is clicked, the UI identifies new `.txt` or `.py` files and prompts the user to "Register" them into the manifest with metadata.
- **Script Run Modal:** Clicking "Run" triggers the `entry.py` CLI logic, capturing the execution logs in a pop-up terminal window.

## Visual Design
- **IDE-Inspired Layout:** Uses a classic sidebar + detail view configuration common in code editors (Visual Studio Code style).
- **Status Badges for Scripts:** Clear indicators for script status (e.g., "Ready", "Missing Dependencies", "Needs Re-registration").
- **Metadata Tooltips:** Hovering over a script entry shows its "Studio" membership and last run-time.
