# Narrative Scripts UI Mockup Project - Master Todo List

**Version:** 0.1.0
**Status:** Draft
**Owner:** TBD
**Last Updated:** 2026-02-04


**Project Goal:** Transform all Narrative Scripts into web-based UIs with dual-mode generation:
- **AI Mode:** User provides context in textarea → AI fills the form
- **Procedural Mode:** RNG/seed-based generation using tables → fills form automatically

**Target Environment:** React + Tailwind (Tauri desktop wrapper)  
**Deliverable per Script:** 
1. ✅ Questionnaire (determine UI requirements)
2. ✅ Answered questionnaire  
3. ✅ ASCII wireframe
4. ✅ UI explanation document
5. ✅ Sample filled output (narrative mockup)

## Notes & Conventions

### File Naming for Outputs
- **Questionnaires:** `docs/mockups/questionnaires/<script-name>-questionnaire.md`
- **ASCII Wireframes:** `docs/mockups/wireframes/<script-name>-wireframe.txt`
- **UI Explanations:** `docs/mockups/ui-docs/<script-name>-ui-explanation.md`
- **Sample Outputs:** `docs/mockups/samples/<script-name>-sample.md`

### Checklist Symbol Key
- `[ ]` = Not started
- `[~]` = In progress
- `[x]` = Completed
- `[!]` = Blocked/needs attention

### RNG & Seed Strategy (Default 80% cases)
- Seed input field (numeric or string)
- "Generate" button triggers deterministic RNG
- Tables loaded from JSON/YAML
- Weighted probabilities when applicable

### AI Mode Strategy (Default 20% cases or hybrid)
- Context textarea (multi-line)
- "AI Fill" button
- AI model selector (handled in settings, not in UI mockup)
- Progress indicator during generation

### Save/Load/Export Features (Case-by-case)
- Save to Firestore (JSON blob)
- Load from library/history
- Export as JSON/CSV/Markdown
- Template versioning
