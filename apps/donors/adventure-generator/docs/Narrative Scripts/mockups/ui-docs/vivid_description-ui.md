# UI Explanation: vivid_description

**Version:** 0.1.0
**Status:** Draft
**Owner:** TBD
**Last Updated:** 2026-02-04


## Component Architecture
- **Dual-Pane Transformer:** A split-screen interface that allows the DM to provide the "Bland" baseline and watch the "Vivid" upgrade happen in real-time.
- **Sensory Multi-Select:** A mandatory checklist to ensure the final output hits at least three of the five senses (Sight, Sound, Smell, Taste, Touch), as required by the script.
- **Urgency & Pacing Slider:** A dynamic control that adjusts the prose structure (short, punchy sentences for high urgency; long, flowery sentences for low urgency).
- **Tone & Emotion Palette:** A tool to inject specific feelings (e.g., Awe, Dread, Solace) into the narrative, moving beyond simple physical description.
- **Interactive Instruction Panel:** Generates "Directorial Cues" for the DM, suggesting specific ways to "Encourage Sensory Exploration" (e.g., "Run your fingers along the smooth bark").

## Interaction Logic
- **Action-Word Enrichment:** Automatically replaces weak/passive verbs with "Strong Action Words" to make the description more vibrant.
- **Figurative Language Toggles:** Users can specifically request Similes, Metaphors, or Personification to add depth and meaning to their scenes.
- **Consistency Auditor:** A background check that prevents contradictory sensory details (e.g., smelling "blooming flowers" in "the dead of winter").

## Visual Design
- **Literary & Authorial:** Uses a clean, elegant layout that prioritizes readability and high-quality typography (Serif fonts for the output).
- **Context-Sensitive Styling:** The UI theme can subtly shift colors based on the chosen "Emotion" (e.g., cool blues for Grief/Solace, vibrant oranges for Urgency/Action).
- **Minimalist Tooling:** Keeps the focus on the prose, utilizing unobtrusive sliders and buttons to avoid cluttering the creative workspace.
