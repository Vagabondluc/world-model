# Planet Pulse Dashboard Onboarding Guide

## Overview

The Planet Pulse Dashboard provides a high-level overview of your world's health and status. It's your first stop for understanding the overall state of the planet.

## Learning Objectives

### 🟢 Beginner (Ages 12-15)
- Understand what the Planet Pulse Dashboard shows
- Learn to read world status badges and risk flags
- Identify top pressures affecting the planet
- Navigate to detailed views from the dashboard

### 🟡 Intermediate (Ages 16-18)
- Understand trend lines and how they indicate change over time
- Interpret confidence levels (low/medium/high)
- Use the consequence preview panel
- Run 50-year projections to see future states

### 🔴 Advanced (Ages 19-99)
- Understand the underlying read model and snapshot system
- Interpret deterministic drill-down tables
- Use the dashboard for hypothesis testing
- Understand the relationship between Planet Pulse and domain authority

## Step-by-Step Walkthrough

### Step 1: Opening the Dashboard

1. From the main dashboard, click on **Planet Pulse** in the navigation menu
2. The dashboard will load with a snapshot of the current world state
3. You'll see three main sections:
   - **Status Overview** (top): World status badges and risk flags
   - **Trend Analysis** (middle): Trend lines for key metrics
   - **Action Panel** (bottom): Commands and projections

### Step 2: Reading Status Badges

**🟢 Beginner:**

Status badges tell you at a glance how your world is doing:

| Badge Color | Meaning | Example |
|-------------|---------|---------|
| 🟢 Green | Healthy | "Atmosphere Stable" |
| 🟡 Yellow | Caution | "Temperature Rising" |
| 🔴 Red | Critical | "Mass Extinction Imminent" |

**🟡 Intermediate:**

Each badge shows:
- Status name (e.g., "Climate Stability")
- Current value (e.g., "72%")
- Trend indicator (↑ ↓ →)
- Confidence level (Low/Medium/High)

**🔴 Advanced:**

Status badges are derived from domain snapshots:
```ts
interface WorldSummarySnapshot {
  worldId: string
  tick: number
  summaryStatus: string[]      // Status badges
  riskFlags: string[]          // Risk indicators
}
```

### Step 3: Understanding Risk Flags

**🟢 Beginner:**

Risk flags warn you about potential problems:

- ⚠️ "CO₂ Levels Rising" - Atmospheric carbon dioxide is increasing
- 🌊 "Sea Level Threat" - Coastal areas at risk from rising seas
- 🔥 "Wildfire Risk" - Increased chance of wildfires in some regions

**🟡 Intermediate:**

Risk flags are prioritized by severity:
1. **Critical** - Immediate action required
2. **High** - Action recommended within 100 years
3. **Medium** - Monitor closely
4. **Low** - Informational only

**🔴 Advanced:**

Risk flags are computed from domain metrics:
- Threshold-based triggers
- Rate-of-change analysis
- Multi-factor risk models
- Confidence-weighted assessments

### Step 4: Analyzing Trend Lines

**🟢 Beginner:**

Trend lines show how values change over time:

```
↑ Rising    ↓ Falling    → Stable    ~ Fluctuating
```

**🟡 Intermediate:**

Each trend line shows:
- Historical data (solid line)
- Projected future (dashed line)
- Confidence band (shaded area)
- Key events (markers on the line)

**🔴 Advanced:**

Trend lines are generated from:
- Time-series data from domain snapshots
- Statistical trend analysis
- Projection models (linear, exponential, logistic)
- Uncertainty quantification

### Step 5: Using the Consequence Preview Panel

**🟢 Beginner:**

Before making changes, see what might happen:
1. Click on an action button
2. The preview panel shows likely outcomes
3. Decide whether to proceed or cancel

**🟡 Intermediate:**

The preview panel shows:
- Primary effects (what directly changes)
- Secondary effects (what happens because of the primary)
- Time to effect (how long before changes appear)
- Confidence in prediction

**🔴 Advanced:**

Preview panel uses:
- Deterministic simulation for short-term effects
- Probabilistic models for long-term projections
- Causal chain analysis
- Multi-domain impact assessment

### Step 6: Running Projections

**🟢 Beginner:**

See the future:
1. Click **"Run 50-Year Projection"**
2. Watch the simulation run
3. Compare current state to projected state

**🟡 Intermediate:**

Projection options:
- Time horizon (10, 50, 100, 500 years)
- Scenario selection (baseline, optimistic, pessimistic)
- Confidence level adjustment
- Export results for analysis

**🔴 Advanced:**

Projections are generated by:
- Extrapolating current trends
- Applying known physical laws
- Considering feedback loops
- Using Monte Carlo methods for uncertainty

## Key Concepts

### 🟢 Beginner Concepts

1. **Snapshot** - A picture of the world at one moment in time
2. **Tick** - One unit of time in the simulation
3. **Domain** - A system of the world (climate, biology, civilization)
4. **Metric** - A measurable quantity (temperature, population, CO₂)

### 🟡 Intermediate Concepts

1. **Read Model** - A simplified view of complex data
2. **Deterministic** - Same inputs always produce same outputs
3. **Confidence Level** - How sure we are about a value
4. **Projection** - A prediction of future states

### 🔴 Advanced Concepts

1. **Domain Authority** - The authoritative source of truth for a system
2. **Snapshot Consistency** - All domains synchronized to same tick
3. **Causal Chain** - A sequence of cause-and-effect relationships
4. **Uncertainty Quantification** - Measuring how uncertain predictions are

## Common Tasks

### Task 1: Check Planet Health

1. Open Planet Pulse Dashboard
2. Look at status badges (green = healthy, yellow = caution, red = critical)
3. Review risk flags for warnings
4. Check trend lines for concerning patterns
5. Decide if action is needed

### Task 2: Compare Current vs Projected State

1. Note current status badges and values
2. Click "Run 50-Year Projection"
3. Wait for projection to complete
4. Compare new status badges to original
5. Use consequence preview to understand changes

### Task 3: Investigate a Risk Flag

1. Click on a risk flag
2. View detailed information in drill-down table
3. Check trend lines for related metrics
4. Run projection to see if risk increases or decreases
5. Navigate to detailed view for deeper analysis

## Tips and Tricks

### 🟢 Beginner Tips
- Start with the status badges - they give you the big picture
- Don't panic about red flags - investigate first
- Use projections to see the future before it happens
- Take notes on what you observe

### 🟡 Intermediate Tips
- Pay attention to confidence levels - low confidence means uncertainty
- Compare multiple trend lines to find correlations
- Use the consequence preview before making changes
- Export projection data for deeper analysis

### 🔴 Advanced Tips
- Understand the domain authority boundaries - what can and cannot be changed
- Use deterministic projections for reproducible experiments
- Combine multiple dashboards for comprehensive analysis
- Leverage the drill-down table for root cause analysis

## Troubleshooting

### Issue: Dashboard shows "Loading..." indefinitely

**Solution:**
- Check if the simulation is running
- Verify world state is valid
- Try refreshing the dashboard

### Issue: Projection fails with error

**Solution:**
- Check if required domains are initialized
- Verify snapshot consistency
- Review error message for specific issue

### Issue: Status badges seem incorrect

**Solution:**
- Check confidence level - low confidence means uncertainty
- Verify tick number is current
- Review underlying metrics in drill-down table

## Glossary

| Term | Definition |
|------|------------|
| **Snapshot** | A complete state of the world at a specific tick |
| **Tick** | One unit of simulation time |
| **Domain** | A system or subsystem of the world (climate, biology, etc.) |
| **Metric** | A measurable quantity used to track state |
| **Projection** | A prediction of future world states |
| **Confidence Level** | How certain we are about a value or prediction |
| **Deterministic** - Same inputs always produce same outputs |
| **Read Model** | A simplified view of complex domain data |
